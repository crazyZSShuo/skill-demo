## 部署前准备

### 克隆仓库

```bash
git clone https://github.com/<org>/<repo>.git
cd <repo>
```

### 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，填写以下必填项：

```env
# LLM 配置
LLM_API_KEY=<your-api-key>
LLM_BASE_URL=<api-base-url>
LLM_MODEL=<model-name>

# 嵌入模型配置
EMBEDDING_API_KEY=<your-embedding-key>
EMBEDDING_MODEL=<embedding-model-name>

# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_PASSWORD=<secure-password>
```

> **警告**：生产环境中必须修改所有默认密码，不得将 `.env` 文件提交至版本控制系统。

## 启动服务

### 一键启动（推荐）

```bash
docker compose up -d
```

### 验证服务状态

```bash
docker compose ps
```

所有服务状态应为 `running`：

```text
NAME                STATUS          PORTS
<project>-api       running         0.0.0.0:8080->8080/tcp
<project>-db        running         0.0.0.0:5432->5432/tcp
<project>-worker    running
```

### 查看启动日志

```bash
docker compose logs -f api
```

## 停止与清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止服务并清除所有数据（不可逆）
docker compose down -v
```

## 故障排查

### 服务无法启动

```bash
docker compose logs <service-name>
```

检查端口是否被占用：

```bash
lsof -i :<port>
```

### 数据库连接失败

确认 `DB_HOST` 与 Compose 网络中的服务名一致，容器内部应使用服务名而非 `localhost`。

### GPU 不可用

```bash
# 验证 NVIDIA 驱动
nvidia-smi

# 验证 Docker GPU 支持
docker run --rm --gpus all nvidia/cuda:12.0-base nvidia-smi
```
